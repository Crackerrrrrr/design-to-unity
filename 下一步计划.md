# FunPlayLanhuGameMcp 下一步计划

## 1. 当前状态

当前第一版 MCP 已经完成基础闭环：

- 可以解析蓝湖项目链接和 `detailDetach` 链接。
- 可以列出蓝湖项目设计图。
- 可以指定设计图生成 Design Implementation Packet。
- 可以下载设计整图参考资源。
- 可以下载图层切图资源。
- 可以给每个节点生成 `global_rect`、`local_rect` 和 `unity_rect_hint`。
- 可以生成 Unity Handoff Profile。
- 可以查询节点树、节点详情、资源清单。
- 可以查询切图专用清单 `lanhu_design_get_slices`。
- 可以生成 Unity 执行计划 `lanhu_design_get_unity_plan`。
- 可以查询轻量摘要 `lanhu_design_get_summary`。
- 可以输出语义候选、置信度、理由和 `requires_semantic_review`。
- 可以输出资源角色 `asset_role`、九宫格候选和资源类型特征。
- 节点已经显式带 `parent_id`，方便外部 MCP 按层级创建对象。

已用真实蓝湖项目验证：

- 设计图 `-h-海报分享` 可以下载切图并获取切图位置。

  - 节点数：77
  - 资源数：77
  - 带切图资源节点数：76
  - 下载成功：77
  - 下载失败：0
  - 本地路径重复：0
  - `lanhu_design_get_slices` 返回切图：76
  - `lanhu_design_get_unity_plan` 返回创建节点：76
  - 不包含参考图时 Unity 资源导入数：76
  - 包含参考图时 Unity 资源导入数：77
  - `lanhu_design_get_summary` 可返回节点统计、语义统计、资源角色统计和 Unity readiness
  - 资源角色统计：`text_sprite` 37、`button_sprite` 21、`icon` 13、`panel` 4、`background` 1、`design_reference` 1

## 2. 下一阶段目标

下一阶段重点不是继续扩平台，而是让 Unity MCP 更容易使用我们的输出，并把真实 MCP 调用链路验收干净。

目标是：

> 让 FunPlayLanhuGameMcp 从“能提供设计数据”升级为“能给 AI 和 Unity MCP 一份可执行的落地清单”。

### 3.2 Unity 执行计划工具

已实现工具：

```text
lanhu_design_get_unity_plan(packet_id)
```

用途：

- 给 AI / Unity MCP 一份执行步骤。
- 明确资源导入顺序、节点创建顺序、组件类型和 warnings。
- 降低 AI 自己组织 packet 的成本。

当前返回内容：

```text
1. 导入 Sprite 资源。
2. 创建 ViewRoot。
3. 按 create_nodes 顺序创建节点；同父节点内按 z_index 从大到小。
4. group -> GameObject + RectTransform。
5. image -> GameObject + RectTransform + Image。
6. text -> GameObject + RectTransform + TextMeshProUGUI。
7. shape -> GameObject + RectTransform + Image 或项目自定义方案。
8. 对 button_candidate 只做候选标记，不自动绑定业务。
9. 写入 source metadata，方便重导入。
10. 保存 prefab。
11. 截图验证。
```

结构化返回字段：

```json
{
  "packet_id": "xxx",
  "target": "unity",
  "asset_imports": [],
  "root": {},
  "create_nodes": [],
  "semantic_candidates": {},
  "warnings": [],
  "rules": []
}
```

## 7. 验收标准

### 7.1 切图工具验收

使用 `-h-海报分享` 设计图验证：

- 能返回 76 个带切图节点。
- 每个切图都有 `local_path`。
- 每个切图都有 `global_rect`。
- 每个切图都有 `unity_rect_hint`。
- 本地文件路径不重复。
- 下载失败数为 0。

### 7.2 Unity Plan 验收

对任意 packet 调用 `lanhu_design_get_unity_plan`：

- 返回资源导入列表。
- 返回节点创建列表。
- 节点顺序按 Unity 可执行创建顺序；同父节点内 `z_index` 大的先创建。
- 每个节点有明确组件类型建议。
- 返回 Unity 坐标转换规则。
- 返回 warning 汇总。

### 7.3 Summary 验收

对大 packet 调用 summary：

- 不返回完整节点树。
- 能在较小 token 内说明设计结构。
- 能列出主要按钮、文本、背景、资源和风险。
